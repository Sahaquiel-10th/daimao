# OPC 数据中心独立迁移说明

这个目录是从主项目里的 `admin-web/` 拆出来的独立网页后台项目。

## 这个目录可以单独拿出去

必须保留：

- `src/`：React 后台源码。
- `server/proxy.cjs`：轻量服务器代理，负责登录、会话、上传文件、转发后台 API。
- `package.json` / `package-lock.json`：依赖和启动脚本。
- `.env.example`：环境变量示例。
- `index.html`：Vite 入口。
- `dist/`：最近一次构建产物；如果部署平台会自动执行 `npm run build`，可以重新生成。

不需要随项目迁移：

- `node_modules/`：用 `npm install` 重新安装。
- 小程序目录 `miniprogram/`。
- 云函数目录 `cloudfunctions/`。
- 数据库迁移目录 `database/`。
- OPC 数据中心视觉素材目录。
- NFC 脚本目录 `scripts/`。

## 它依赖哪些外部能力

管理后台不直接连 SQL，也不持有数据库密码。

浏览器/服务器只调用：

- CloudBase 云函数：`daimaoBusiness`
- CloudBase 云存储：用于图片上传

真正的数据读写仍由 `daimaoBusiness` 完成，并由云函数内的权限逻辑控制。

## 本地启动

```bash
npm install
npm run dev
```

## 生产构建

```bash
npm run build
```

## 环境变量

参考 `.env.example`。生产环境通常还需要服务器侧配置后台账号、密码、CloudBase 环境和云函数名。

注意：`ADMIN_WEB_TOKEN`、管理员密码、CloudBase Secret 等敏感配置不要写进前端源码，也不要提交到 Git。

## 和主项目的关系

当前主项目里仍保留一份 `admin-web/`，用于旧部署链路不被打断。

等独立后台确认能正常部署和登录后，可以再决定是否从主项目里移除 `admin-web/`。
