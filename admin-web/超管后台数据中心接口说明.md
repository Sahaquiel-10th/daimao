# OPC 数据中心：超管后台接口说明

> 审计范围：`cloudfunctions/daimaoBusiness/index.js` 与 `cloudfunctions/daimaoPartnerApi/index.js`（只读审计，2026-07-13）。本文列出数据中心当前可调用的全部入口能力，供超管后台规划页面与菜单。

## 1. 超管鉴权与调用方式

超管后台直接调用 `daimaoBusiness`，请求主体为 `{ action, adminWebToken, ...参数 }`。`adminWebToken` 必须等于云函数环境变量 `ADMIN_WEB_TOKEN`；服务端随后选取一个 `status=active,is_admin=1` 的用户（若配置 `ADMIN_WEB_OPENID` 则固定该管理员），各 `admin*` handler 再执行 `requireAdmin`。

风险提示：这是一枚共享静态高权限令牌。只能由受控后台服务端持有，不应写入网页源码、小程序包、本地存储或日志。更理想的生产方案是后台登录会话 + 短期令牌 + 操作人绑定；当前审计仅记录代码现状。

统一响应：成功 `{"success":true,...}`；失败 `{"success":false,"code","message"}`。大多数管理写操作会写 `admin_logs`。

## 2. 建议的超管后台功能模块

### 2.1 总览与通用查询

- `healthCheck`：数据库、核心表、AI/向量服务健康状态（该入口在鉴权分支之前，代码现状无需管理员令牌）。
- `adminList`：通用后台聚合列表/总览数据。
- `publicDashboardStats`：数据大屏统计；若配置 `DASHBOARD_PUBLIC_TOKEN`，传 `dashboardToken`。
- `publicProjectTown`：公开项目大屏；同上。

### 2.2 用户与权限

- `adminListUsers`：用户列表；`limit`、`keyword`。
- `adminSetUserStatus`：启用/停用用户；`userId`、`status`。
- `adminSetUserAdmin`：授予/移除管理员；`userId`、`isAdmin`；不能移除当前管理员自身权限。
- `adminReviewCandidate`：审核候选对象；`targetType`、`targetId`、`status`。

### 2.3 社区徽章与成员认证

- `adminListCommunityBadges`：`communityId`，列出社区徽章。
- `adminUpsertCommunityBadge`：新增/更新徽章；`badgeId|id`、`communityId`、`badgeName`、`logoUrl`、`description`、`status`、`sortWeight`。
- `adminSetCommunityMemberBadge`：为成员设置徽章；`communityId`、`userId|targetUserId`、`badgeId`。
- `adminCreateCommunityMemberEvidence`：创建社区成员认证证据；`communityId`、`userId`、`evidenceType`、`tags`。

### 2.4 项目管理

- `adminListProjects`：项目列表；`limit`。
- `adminCreateProject`：创建项目；`project|patch`。
- `adminUpdateProject`：更新项目；`projectId`、`patch`。
- `adminGetProjectManagement`：项目管理详情；`projectId`。
- `adminUpsertProjectMember`：新增/更新成员；`projectId`、`userId|memberUserId`、`member|patch`。
- `adminCreateProjectUpdate`：新增项目动态；`projectId`、`update`。
- `adminUpdateProjectUpdate`：更新项目动态；`projectId`、`updateId`、`update`。
- `adminCompleteProject`：完成项目；`projectId`、`summary|completionSummary`、`updateTitle`、`visibility`、`memberReviews`。
- `adminCreateProjectMemberReview`：项目成员评价；`projectId`、`reviewedUserId|userId`、可选 `reviewerUserId`、`review`、`tags`。

### 2.5 项目申请审核

- `adminListProjectApplicationReviews`：审核列表；`statuses[]`、`limit`。
- `adminGetProjectApplicationReview`：审核详情；`applicationId`。
- `adminDecideProjectApplication`：人工决定；`applicationId`、`decision`、`feedback`。
- `processProjectApplicationReviews`：处理待 AI 审核任务；后台管理员可手动触发，定时触发需 `schedulerSecret`。
- `processProjectApplicationReviewDeadlines`：处理审核超时；同上。

### 2.6 活动管理

- `adminListEvents`：活动列表；`limit`。
- `adminCreateEvent`：创建活动；`event`（标题、描述、类型、地点、开始/结束、社区、状态、可见性、费用等）。
- `adminUpdateEvent`：更新活动；`eventId`、`event`。
- `adminConfirmEventRegistration`：确认报名付款；`eventId`、`externalPaymentNo|paymentNo`、`paidAmountCents`、`paidAt`、`note`。
- `processEventReminders`：处理活动提醒；计划任务入口，需相应定时任务参数/密钥。

### 2.7 用户证据与 RAG

- `adminCreateUserEvidence`：创建用户证据；`userId|targetUserId`、可选 `projectId/eventId/communityId`、`evidenceType`、`file/content/evidence`、`confidence`、`title`、`tags`。
- `adminListUserEvidence`：用户证据列表；`userId|targetUserId`。
- `adminUpdateUserEvidence`：更新证据；`evidenceId|id` 及可修改字段。
- `adminArchiveUserEvidence`：归档证据；`evidenceId|id`。
- `adminEnsureUserRagTag`：确保/重建用户 RAG 标签；`userId`、`force|recreate`、标签参数、`dryRun`、`confirm`。
- `adminRequeueUserRagIndex`：重排用户索引任务；`userId`、`limit`。
- `processRagIndexJobs`：执行 RAG 索引队列；管理员手动调用或用 `schedulerSecret` 定时调用。

### 2.8 REAI 向量库运维

- `adminReaiGetProject`：读取 REAI 项目配置。
- `adminReaiProjectTagPatch`：修改项目标签；`operation`、标签参数、payload、`dryRun`、`confirm`。
- `adminReaiPatchVdbTags`：修改向量文档标签；`objectId|vectorDocId`、`tags`、`content`、`dryRun`、`confirm`。

### 2.9 外部应用（社区小程序）管理

- `adminUpsertAppClient`：新增/更新接入应用；`client`。关键字段包括 appid、状态、绑定社区及 `allowedActions`。
- 外部网关实际入口为 `daimaoPartnerApi`；默认允许 action 见外部小程序文档。

安全要求：后台 UI 对 `allowedActions` 使用固定安全枚举和分组勾选，不允许自由输入；至少明确禁止 `admin*`、调试、定时任务和内部项目编辑 action。

### 2.10 外部应用钱包、充值与计费

- `adminGetAppClientBilling`：计费详情/流水；`appClientId|appid`、`startAt/endAt`、`page/pageSize`。
- `adminCreateAppClientRecharge`：创建充值单；应用定位参数 + `amountCents|rechargeUnits`、`status`、`currency`、支付渠道、外部单号、备注。
- `adminConfirmAppClientRecharge`：确认充值；应用定位参数 + `orderId`、`externalPaymentNo`。
- `adminAdjustAppClientBalance`：人工调账；应用定位参数 + `mode/entryType/category`、金额或单位、`reason`、幂等键、凭证引用。
- `adminGetPlatformBillingSettings`：读取平台计费参数。
- `adminUpdatePlatformBillingSettings`：更新平台计费参数；`settings`。
- `adminUpdateAppClientBillingSettings`：更新单应用计费设置；应用定位参数 + `settings`。
- `adminSetAppClientWalletStatus`：冻结/启用钱包；应用定位参数 + `status`、`reason`。
- `adminRotateAppClientBillingReadToken`：轮换社区计费只读令牌；应用定位参数。新令牌只应显示一次。
- `communityGetAppClientBilling`：社区使用只读令牌查看自身计费；这是合作方后端入口，不属于超管写操作。

## 3. 普通业务 action（超管也可调用）

超管身份仍可使用以下前台/成员能力。后台若要提供“代运营/业务观察”页面，可按业务角色限制接入。

### 用户资料与发现

`getMyIdentity`、`getProfileSchema`、`saveMyProfile`、`listCommunityUsers`、`getCommunityUserProfile`、`getAgentProfile`、`saveAgentProfile`、`getAssistantContext`、`listRecommendedUsers`、`listMyReferredUsers`、`getRecommendations`、`listNotifications`、`markNotificationRead`。

注意：`listCommunityUsers` 和 `getCommunityUserProfile` 当前会返回全中心活跃用户，且可见资料含微信号；只适合严格受控超管页面，不应作为社区通讯录直接复用。

### 项目全流程

`listProjects`、`listMyProjects`、`getProject`、`applyProject`、`toggleWatch`、`publishUpdate`、`updateProjectUpdate`、`updateProjectProgress`、`createMeetingRequest`、`respondMeetingRequest`、`respondProjectApplication`、`respondProjectContactConsent`、`inviteMember`、`acceptProjectInvitation`、`getProjectSpace`、`createProjectRecord`、`analyzeProjectRecord`、`confirmReminderIntent`、`createProjectEvent`、`authorizeReminder`。

### 活动、AI 秘书与撮合

`listEvents`、`registerEvent`、`runAiTask`、`listSecretaryChats`、`startSecretaryChat`、`advanceSecretaryChatRun`、`continueSecretaryChat`、`retrySecretaryChatRun`、`sendSecretaryContactCard`、`endSecretaryChat`、`getSecretaryChat`、`markSecretaryChatRead`、`requestSecretaryMeet`、`respondSecretaryMeet`。

这些接口中的“当前用户”在网页令牌模式下是 `ADMIN_WEB_OPENID` 指定的管理员（未指定则取首个活跃管理员），不是后台正在查看的任意用户。不要把它们误做成无约束的“代用户操作”。

## 4. 调试与测试 action（生产后台默认隐藏）

以下接口仅在 `ENABLE_DEBUG_ACTIONS=true` 时可用，生产菜单应隐藏并关闭该环境变量：

- `debugEcho`：运行环境回显。
- `adminDebugDirectSqlSmoke`：数据库烟测；需网页令牌。
- `testAiConnection`：AI 连通测试；需 `confirm=test-ai-connection`。
- `seedDemoData`：写入演示数据。
- `adminDebugProjectApplicationInput`：构造申请审核输入。
- `adminTestAppClientAiBilling`：外部应用 AI 计费测试。
- `adminTestSingleEvidenceSearch`：单条证据检索测试。
- `adminTestProjectApplicationEvidence`：申请证据检索测试。
- `adminTestProjectApplicationReview`：申请 AI 审核测试。

## 5. 系统/计划任务入口（不做人工业务按钮）

- `sendDueReminders`：发送到期提醒；必须传正确 `schedulerSecret`。
- `processRagIndexJobs`：RAG 队列；定时模式需要 `schedulerSecret`。
- `processProjectApplicationReviews`：申请 AI 审核队列；定时模式需要 `schedulerSecret`。
- `processProjectApplicationReviewDeadlines`：申请审核截止处理；定时模式需要 `schedulerSecret`。
- `processEventReminders`：活动提醒计划任务。

计划任务密钥不得进入浏览器。后台如需“立即执行”，应由后台服务端代发并记录操作审计。

## 6. 前端菜单建议与危险操作确认

- 一级模块：总览、用户、社区认证、项目、申请审核、活动、证据/RAG、外部应用、计费钱包、系统运维。
- 写操作统一显示目标 ID/名称和影响范围；停用用户、管理员变更、完成项目、归档证据、钱包调账、冻结钱包、令牌轮换、REAI 修改需二次确认。
- 充值确认、余额调账必须要求幂等键或外部单号，并展示操作后的余额与流水 ID。
- `dryRun` 能力先预览再确认；`confirm` 字符串由后端规则生成/校验，不在前端自动填充。
- 超管页面不要展示 openid、内部密钥、令牌哈希、完整 AI/RAG 原文，除非具体运维功能确有需要。
